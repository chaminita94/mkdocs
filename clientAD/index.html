Unió d’equips Windows al domini AD

Unió d’equips Windows al domini Active Directory

En aquest apartat configurarem un equip client amb Windows i el connectarem al servidor Active Directory que hem configurat prèviament.

El primer que farem és comprovar la configuració de xarxa del client. Si l’equip mostra una IP del tipus 169.x.x.x, vol dir que no ha rebut cap IP vàlida i està utilitzant una IP APIPA, és a dir, una adreça automàtica per defecte. Això passa perquè encara no tenim un servidor DHCP actiu.

Per solucionar-ho, configurarem una IP estàtica manualment al client. És molt important que el servidor DNS apunti a la IP del servidor Active Directory, ja que aquest també actua com a servidor DNS.

Un cop assignada la IP, comprovem la connectivitat entre el client i el servidor, i viceversa, utilitzant comandes ping.

Si la comunicació no funciona, pot ser a causa del Tallafoc (Firewall). Pots optar per desactivar-lo temporalment per a aquesta pràctica o bé habilitar les regles necessàries per permetre el trànsit entre el servidor i el client.

Per iniciar la connexió del client al domini, escrivim nom a la barra de cerca de Windows i seleccionem l’opció Canvia el nom del grup de treball o de l’equip.

A la finestra que s’obre, fem clic a Canvia..., a la secció que diu: Per canviar el nom d’aquest equip o el domini al qual pertany.

Abans de continuar, recorda el nom exacte del teu domini. Abans d’introduir-lo, pots fer una prova de connexió mitjançant ping des del CMD, per exemple: ping vitaliy.asix. Si reps resposta, significa que la resolució DNS i la connexió funcionen correctament.

Introdueix el nom del domini a la finestra i prem D'acord.

Tot seguit, se’t demanarà introduir un usuari amb permisos d’administrador del domini. Introdueix el nom d’usuari i la seva contrasenya. (Recorda que pot variar segons la configuració del teu entorn).

Després d’uns instants, si tot és correcte, apareixerà un missatge de benvinguda al domini. A continuació, reiniciarem l’equip.

Ara bé, com que encara no hem creat cap usuari de domini, de moment no podrem iniciar sessió com a tal. La creació d’usuaris i unitats organitzatives (OUs) es tractarà en un apartat posterior.
