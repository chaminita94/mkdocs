    Permisos Especials i ACL

    Permisos Normals UOG

    Els permisos "normals" en Linux es gestionen amb les comandes chown, chgrp i chmod. Aquesta taula mostra les comandes principals i el que fan

    Comanda Descripció
    chown [-R] Canvia el propietari d’un fitxer o carpeta.
    chgrp [-R] Canvia el grup propietari d’un fitxer o carpeta.
    chmod [-R] Modifica els permisos d’un fitxer o carpeta.

    Modes de permisos

    Cada fitxer o carpeta pot tenir permisos assignats per l'usuari (u), el grup (g) i altres (o). Aquesta taula defineix els valors dels permisos

    Mode Permisos
    0 ---
    1 --x
    2 -w-
    3 -wx
    4 r--
    5 r-x
    6 rw-
    7 rwx

    UMASK

    La umask és un valor que defineix els permisos predeterminats dels fitxers i directoris nous. Aquesta taula mostra com s'apliquen aquests valors:

    Configuració Directori Fitxers
    Original 777 666
    Umask 022 022
    Resultat 755 644

    Explicació:
    Sense cap canvi en la configuració predeterminada de la umask, tots els fitxers creats per l'usuari root obtindran permisos 644 (666 - 022), i tots els directoris obtindran permisos 755 (777 - 022).

    Com funciona la umask?

    La umask no funciona per resta aritmètica, sinó bit a bit, apagant (desactivant) els permisos corresponents. Cada bit del valor predeterminat interacciona amb el bit equivalent de la umask.

    Exemple de càlcul per a fitxers

    Per calcular els permisos per a fitxers, seguim aquests passos:

    Pas Permisos Binari
    Predeterminats 666 110 110 110
    Umask 022 000 010 010
    NOR de l'Umask Negació 111 101 101
    Resultat (AND) Càlcul final 110 100 100

    Els permisos resultants són 644: - Propietari: pot llegir i escriure. - Grup i altres: només poden llegir.

    Exemple de càlcul per a directoris

    Per calcular els permisos per a directoris, seguim aquests passos:

    Pas Permisos Binari
    Predeterminats 777 111 111 111
    Umask 022 000 010 010
    NOR de l'Umask Negació 111 101 101
    Resultat (AND) Càlcul final 111 101 101

    Els permisos resultants són 755: - Propietari: pot llegir, escriure i executar. - Grup i altres: només poden llegir i executar.

    Explicació Visual del Procés

    1. NOR (Negació): S'inverteixen els bits de l'Umask.
      Exemple: ~000 010 010 → 111 101 101.
    2. AND (Intersecció): Es combinen els permisos predeterminats amb el resultat de la NOR.
      Exemple: 110 110 110 AND 111 101 101 → 110 100 100.
    Tipus Original Umask Resultat
    Fitxers 666 (110 110 110) 022 (000 010 010) 644 (110 100 100)
    Directoris 777 (111 111 111) 022 (000 010 010) 755 (111 101 101)

    Permisos Especials

    Els permisos especials com sticky, suid i sgid permeten funcionalitats addicionals en fitxers o carpetes. La taula següent explica cadascun d'ells:

    Permís Comanda Descripció
    sticky chmod 1 +t Bloqueja la modificació d’arxius per altres usuaris.
    suid chmod 4 u+s Executa amb privilegis del propietari.
    sgid chmod 2 g+s Executa amb privilegis del grup.

    ACLs (Access Control Lists)

    Les ACLs permeten definir permisos més granulars que els permisos "normals". Aquestes comandes permeten gestionar-les:

    Comanda Descripció
    getfacl Mostra les ACLs del fitxer/carpeta.
    setfacl -m Defineix permisos ACL.
    setfacl -b Esborra les ACLs del fitxer/carpeta.
