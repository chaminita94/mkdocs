    Comandes per a la gestió de paquets en sistemes Debian/Ubuntu

    apt-get o apt

    apt és un front-end per a dpkg amb millores respecte a apt-get, facilitant la gestió de paquets i el manteniment del sistema.

    Comandes Bàsiques d'apt

    • apt update
      Actualitza la llista de tots els paquets mirant el fitxer /etc/apt/sources.list.
      Exemple: Després d'executar sudo apt update, hauries de veure un missatge que indica que la llista de paquets s'ha actualitzat correctament, amb informació sobre paquets que s'han actualitzat.

    • apt upgrade
      Actualitza el sistema instal·lant noves versions de paquets existents, però no instal·la nous paquets.
      Exemple: Si executeu sudo apt upgrade, es mostrarà una llista de paquets que seran actualitzats, amb la quantitat d'espai que ocuparan.

    • apt dist-upgrade
      Igual que l'anterior, però també instal·la nous paquets si cal, i pot eliminar paquets si és necessari per resoldre dependències.
      Exemple: sudo apt dist-upgrade és útil quan s'actualitzen les versions del nucli o altres components importants del sistema.

    • apt install paquet
      Instal·la un paquet específic.
      Exemple: sudo apt install vim instal·larà l'editor de text Vim.

    • apt remove paquet
      Elimina un paquet instal·lat, però manté els fitxers de configuració.
      Exemple: sudo apt remove vim eliminarà Vim, però els fitxers de configuració es mantindran.

    • apt purge paquet
      Elimina un paquet juntament amb els seus fitxers de configuració.
      Exemple: sudo apt purge vim eliminarà Vim i tots els fitxers associats.

    • apt autoremove
      Esborra els paquets que ja no són necessaris ni utilitzats pel sistema.

    • apt clean
      Esborra els paquets descarregats que es troben en la memòria cau (cache).

    • apt-cache depends paquet
      Mostra les dependències d'un paquet determinat.

    • apt-rdepends paquet
      Mostra les dependències de les dependències d'un paquet (en forma recursiva).

    • apt -f install
      Corregeix instal·lacions a mitges i repara paquets trencats.

    • apt-cache policy paquet
      Mostra les versions disponibles d'un paquet i el seu estat, incloent-hi informació sobre el "pinning".

    • Fitxers de configuració:
      Es pot configurar la política de "pinning" a través del fitxer /etc/apt/preferences.d/.

    aptitude

    aptitude és una altra eina per gestionar paquets que inclou una interfície gràfica i és més sofisticada per gestionar dependències.

    • aptitude remove paquet
      Elimina un paquet instal·lat, igual que apt remove.

    • aptitude purge paquet
      Elimina un paquet i els seus fitxers de configuració, igual que apt purge.

    • aptitude install paquet
      Instal·la un paquet específic.

    • aptitude show paquet
      Mostra informació detallada sobre un paquet, i si no està instal·lat, ho indica.

    dpkg

    dpkg és l'eina bàsica per a la gestió de paquets en sistemes Debian, però és més baixa a nivell que apt o aptitude.

    • dpkg -r paquet
      Elimina un paquet instal·lat, però conserva els fitxers de configuració.

    • dpkg -P paquet
      Elimina un paquet i també els seus fitxers de configuració.

    • dpkg -i arxiu.deb
      Instal·la un paquet des d'un fitxer .deb local.

    • dpkg -s paquet
      Mostra informació sobre un paquet instal·lat, incloent-hi la seva prioritat (e.g., required, important, standard, etc.).

    • dpkg-reconfigure
      Reconfigura un paquet ja instal·lat.

    • dpkg --get-selections | grep paquet
      Mostra l'estat d'un paquet; no el mostra si no està instal·lat.

    Repositoris

    Els repositoris són col·leccions de paquets on el sistema obté el programari. Aquests es configuren en el fitxer /etc/apt/sources.list.

    • main
      Programari suportat oficialment per la distribució.

    • restricted
      Programari suportat però amb llicències no completament lliures.

    • universe
      Programari suportat per la comunitat.

    • multiverse
      Programari no lliure, però disponible en els repositoris.

    Com afegir repositoris

    Aquest document explica com afegir nous repositoris al sistema APT en sistemes basats en Debian o Ubuntu.

    Editar el fitxer sources.list

    El fitxer /etc/apt/sources.list és on es defineixen els repositoris des d'on APT obté els paquets. Per afegir un nou repositori:

    Obre el fitxer amb un editor de text, amb privilegis d'administrador:

    sudo nano /etc/apt/sources.list

    També pots crear un nou fitxer:

    sudo nano /etc/apt/sources.list.d/google.list.

    Afegeix aquesta línia per afegir el repositori de Google Chrome:

    deb [arch=amd64] http://dl.google.com/linux/chrome/deb/ stable main

    Actualitza la base de dades de paquets per incloure el repositori afegit:

    sudo apt update

    Per instal·lar Google Chrome, utilitza la comanda següent:

    sudo apt install google-chrome-stable

    Afegir la clau GPG de Google

    Per verificar els paquets descarregats de Google, és necessari afegir la clau pública. Això es pot fer amb:

    Afegir la clau pública: 

    wget -q -O - https://dl.google.com/linux/linux_signing_key.pub | sudo apt-key add -

    Afegir el repositori de Google

    Abans d'instal·lar Google Chrome, necessitaràs afegir el repositori de Google a les fonts d'APT. Això es pot fer amb:

    Afegir el repositori: 

    echo "deb [arch=amd64] https://dl.google.com/linux/chrome/deb/ stable main" | sudo tee /etc/apt/sources.list.d/google-chrome.list

    Actualitzar la llista de paquets

    Un cop afegit el repositori, és necessari actualitzar la llista de paquets. Això es pot fer amb:

    Actualitzar la llista: 

    sudo apt update

    Instal·lar Google Chrome

    Ara pots instal·lar Google Chrome amb la següent comanda:

    Instal·lar Google Chrome: 

    sudo apt install google-chrome-stable

    Compilar a través de codi font

    Abans de compilar un programa des del codi font, cal tenir un compilador instal·lat al sistema. Una opció comuna és el paquet build-essential, que es pot instal·lar amb la següent comanda:

    apt-get install build-essential

    Instal·la el paquet build-essential, que conté eines necessàries per compilar programari.

    Compilar el codi font

    Un cop dins de la carpeta, es poden executar les següents comandes per compilar el programa:

    • Configuració del programa
      ./configure
      Prepara el sistema per a la compilació del programa, verificant requisits i generant fitxers de configuració.

    • Compilar el programa
      make
      Inicia el procés de compilació del codi font, generant executables o binaris.

    • Instal·lar el programa
      make install
      Instal·la el programa al sistema, copiant els fitxers necessaris als directoris apropiats.

    • Desinstal·lar el programa
      make uninstall
      Desinstal·la el programa, eliminant els fitxers instal·lats anteriorment.

    Comandes per a descomprimir arxiusa

    • Arxiu .tar
      tar xvf archivo.tar
      Descomprimeix un arxiu amb extensió .tar. x indica que volem extreure, v mostra els noms dels arxius extrets, i f especifica el fitxer.

    • Arxiu .gz
      gunzip archivo.gz
      O bé gzip -d archivo.gz
      Descomprimeix un arxiu .gz. gunzip és la manera específica de fer-ho, mentre que gzip -d és una forma general per descomprimir.

    • Arxiu .tgz o .tar.gz
      tar xvzf archivo.tar.gz
      Descomprimeix arxius amb format .tar.gz o .tgz. z indica que el fitxer està comprimit amb gzip.

    • Arxiu .bz2
      bunzip2 archivo.bz2
      O bé bzip2 -d archivo.bz2
      Descomprimeix arxius .bz2. bunzip2 és un mètode més directe, mentre que bzip2 -d és una forma general.

    • Arxiu .tar.bz2
      tar xvjf archivo.tar.bz2
      Descomprimeix arxius amb extensió .tar.bz2. j indica que s'utilitza el mètode de compressió bzip2.

    • Arxiu .zip
      unzip archivo.zip
      Descomprimeix fitxers en format .zip.

    • Arxiu .rar
      rar -x archivo.rar
      Descomprimeix arxius .rar. -x indica que volem extreure els fitxers.

    Entrar a la Carpeta

    Un cop descomprimits els fitxers, és necessari accedir a la carpeta creada. Això es pot fer amb:

    Accedir a la carpeta: cd nom_de_la_carpeta
    Exemple: cd projecte

    Sortir de la carpeta: cd ..
    Torna al directori pare.
    Exemple: Si estàs a projecte, cd .. et portarà al directori anterior.

    Tornar al directori anterior: cd --
    Exemple: Si has canviat de ~/documents a ~/documents/projecte, cd -- et portarà de nou a ~/documents.

    Notes: - cd és sensible a majúscules i minúscules. - Utilitza cd .. per navegar cap enrere en l'estructura de carpetes. - cd -- és útil per alternar entre dues carpetes sense recordar la ruta completa.

    Les comandes cd nom_de_la_carpeta, cd .. i cd -- són essencials per navegar pel sistema de fitxers i accedir a les carpetes descompresses.
